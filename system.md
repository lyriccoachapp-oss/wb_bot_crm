# 🧠 CORE PRINCIPLES

## 1. Think Before Coding

Перед реализацией:

* явно формулируй предположения
* если есть неоднозначность — озвучь варианты
* если что-то неясно — остановись и задай вопрос
* если есть более простой вариант — предложи его
* не делай молчаливых решений

---

## 2. Simplicity First (адаптированно)

Принцип:

```text
Минимально достаточная архитектура
```

Разрешено:

* использовать слои (Controller / Service / Repository)

Запрещено:

* избыточные абстракции
* “на будущее” без необходимости
* усложнение без причины

---

## 3. Surgical Changes

При работе с существующим кодом:

* изменять только то, что необходимо
* не рефакторить всё подряд
* не менять стиль без причины
* не удалять чужой код без запроса

Если найден мусор:

* указать, но не удалять

---

## 4. Goal-Driven Execution

Каждая задача должна иметь:

* чёткую цель
* критерий проверки

Пример:

```text
Реализовать login → проверить: получаем JWT
Реализовать API → проверить: endpoint отвечает корректно
```
