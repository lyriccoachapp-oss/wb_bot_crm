# 🔧 ДОПОЛНЕНИЕ К ЗАДАНИЮ: СТАНДАРТЫ И ТРЕБОВАНИЯ К API

## 🧠 Общий подход

Проект должен строиться по принципу:

```text
API-first architecture
```

## 🌐 Стандарты API

### 📌 1. Версионирование (обязательно)

Все endpoints должны иметь версию:

```http
/api/v1/...
```

---

### 📌 2. Соответствие REST

API должно соответствовать принципам REST:

* ресурсы через существительные:

  * `/users`
  * `/receipts`
  * `/objects`

### Микросервисы (Docker Compose)
1.  **Web** (`crm_web`, порт `8073`):
    *   Nginx-сервер, отдающий статику (JS, CSS) и PHP-шаблоны без логики БД.
2.  **API** (`crm_api`, порт `8072`):
    *   Основной бэкенд (Laravel 11). Отвечает за логику (JWT-аутентификация, сохранение данных, Telegram-webhook, управление очередями).
3.  **Bot** (`crm_bot`, порт `8071`):
    *   Webhook-обработчик для Telegram. Маршрутизирует сообщения и вызывает методы API.
4.  **Worker** (`crm_worker`):
    *   Фоновый воркер (Laravel Queue). Обрабатывает тяжелые задачи (например, фоновое распознавание чеков) асинхронно, используя таблицу `jobs`.
5.  **OCR** (`crm_ocr`, порт `8074`):
    *   Локальный микросервис на Python (PaddleOCR). Распознает текст с картинок для воркера чеков. должны иметь единый формат:

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

Ошибки:

```json
{
  "success": false,
  "error": "Error message"
}
```

---

### 📌 4. Пагинация

Все списки должны поддерживать пагинацию:

```http
GET /api/v1/receipts?page=1&limit=20
```

```php
    $phpmailer->Port       = 465;
```
